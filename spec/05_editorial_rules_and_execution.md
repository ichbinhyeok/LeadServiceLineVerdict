# 05 Editorial Rules And Execution

## 1) Editorial rules
- Lead with the inventory answer or the decision answer.
- Use exact utility naming and exact program naming.
- Separate the service line problem from interior plumbing and fixture lead.
- Label estimates as estimates and source facts as source facts.
- Keep every local page timestamped.

## 2) Trust rules
- This is public-health-adjacent content.
- Never imply that filter use removes the need for replacement.
- Never promise utility funding.
- Never generalize one utility's inventory to another utility.
- Always link to the official utility or EPA page when available.

## 3) Style rules
- Use short sentences on notice and transaction pages.
- Avoid jargon unless the page is for contractors or inspectors.
- Make the next step visually obvious.
- Put caution language near any estimate or program claim.

## 4) Delivery phases
### Phase 1
- Lock the schema-validated `CSV` to `JSON` data pipeline.
- Build the utility registry.
- Launch 10-15 utility pages in two states with clear inventory data.
- Publish the evergreen explanation pages.

### Phase 2
- Add replacement-cost pages for the first cluster of utilities.
- Add filter and testing pages.
- Add buyer/seller pages where transaction intent is strong.

### Phase 3
- Expand the program registry.
- Add state hubs and cross-utility comparison pages.
- Start sponsor outreach after the data model is stable.

## 5) Success metrics
- Indexed utility pages
- Pages with verified source links
- CTR from inventory and notification pages
- quote or consult conversions from cost pages
- sponsor conversion rate
- stale page rate

## 6) Definition of done for launch
- 10+ utility pages with official source links and verification dates
- 3+ evergreen guides published
- a repeatable source-verification workflow
- a documented `CSV` to normalized `JSON` build pipeline
- cost, program, filter, and buyer/seller page templates stable
- one sponsor placement format defined for local providers
